BUFFER

Global Game Jam 2018 Entry (Theme: Transmission) January 2018
Python/Pygame
(Video sound is a little high, lowering volume might be needed. sorry)

Rules:

  • There are nodes, and connections.
  • There is a starting node, and a target node.
  • The starting node starts with the 'matter' - which needs to be transmitted to the target node.
  • A node can only transmit to another node if there is connection between them.
  • Nodes can only hold the matter for some time.
  • The connections randomly appear and disappear.
  • Transmit the matter before the target times out or matter is lost!

Intergalactic Dance Robot

Game Design & Analysis Class Project, November - December 2017
Unity/C#
Team: Alex Coburn, Barru Tolety, Kyros Jalife, Erinc Argimak

Rules:

  • Each limb of the robot can be controlled with a button.
  • The dance move holograms come towards the robot.
  • When the hologram touches the robot, the robot should have the same pose.
  • The robot loses charge over time.
  • Some limbs of the robot have 'orbs' - that charge the robot for some energy.
  • Survive until the end of the round and score as high as possible!

Under the Sea

Game Design & Analysis Class Project, September - November 2017
Board Game
Team: Xiao Yu, Anqi Liu, Zhu Ge, Wenqi Fan, Erinc Argimak

Rules:

  • Players are controlling teams of monsters.
  • At the center of the board is a treasure.
  • The goal is to get the treasure and bring it back to the spawn.
  • The middle, darker area makes whoever enters invisible, so players should guess where their opponents' monsters are and reveal them to be able to capture them.
  • Each turn, each monster can move maximum 3 steps, and use their skill only once.
  • Monsters have capturing sides and weak sides, they can capture each other.
  • Captured monster is sent back to its spawn point.
  • The board includes various actions such as teleport, or reveal.

Colors

Idea Sprout: May 2014
First Prototype: May 2014, Python / PyGame
Web Prototype: April 2017 (Updated February 2018), HTML5 / JS / Phaser

Rules:

  • Cell values: 1-6, Cell Neighbors: Left, Right, Top, Bottom
  • Neighbor has (own value + 1) -> Cell can be incremented short press & release
  • 1 -> 2 -> 3 -> 4 -> 5 -> 6 -> 1
  • Cell cluster with same value can all change at once. long press & release on a changable cell
  • CHANGING CLUSTERS COUNTS FOR A SINGLE MOVE
  • Goal: Turn all cells into the same color in least amount of moves.
